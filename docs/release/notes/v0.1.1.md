# Review Agent CLI v0.1.1

This patch release completes the installable CLI's AI SDK 7 migration and
tightens its native execution boundary:

- updates `ai`, AI Gateway, and the OpenAI-compatible provider to their latest
  compatible AI SDK 7 patches while keeping private review telemetry disabled;
- disables AI SDK retries so provider routing policy owns CLI model attempts;
- counts native helper output incrementally instead of rescanning accumulated
  output;
- validates native diff output against the generated chunks-only contract and
  derives patch and changed-line indexes from that single source; and
- pins Rust 1.96.1 for release builds and refreshes the pinned GitHub Actions
  used by project CI and the consumer workflow.

## Install and verify

Download the archive, manifest, and adjacent checksum files for your platform
from this release. Verify both checksums before extracting. The complete command
sequence and platform matrix are in the
[CLI distribution guide](https://github.com/BjornMelin/agent-review/blob/v0.1.1/docs/release/cli-distribution.md).

- Requires Node.js 24 or newer and git.
- Supports glibc 2.39+ Linux (`x64`, `arm64`), macOS (`x64`, `arm64`), and
  Windows (`x64`).
- Does not support Windows arm64, musl Linux, or glibc versions older than 2.39.
- Emits deterministic JSON, SARIF 2.1.0, and Markdown reports with stable exit
  codes `0` through `4`.
- Source commit: `{{SOURCE_SHA}}`.

Every native archive is built and smoke-tested on its target runner. The
release workflow verifies per-asset SHA-256 files, combines them into
`SHA256SUMS`, downloads the draft assets, byte-compares the readback, and only
then publishes the release.

## Known constraints

Provider-backed reviews still require the corresponding Codex, AI Gateway, or
OpenRouter credentials. The hosted service and Review Room remain
operator-managed deployment surfaces and are not a public SaaS offering.
